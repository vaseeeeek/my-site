  • Типы переменных
    23.08.2022

    • Приметивные типы переменных

      string, boolean, number, null, undefined, symbol

      Значение приметивных типов хранит само значение, а не ссылку на значение как object

    • Ссылочный тип

      object(обьект)

      Переменная хронящая обьект будет хранить только ссылку на обьет а не сам обьект

      Поменяв значение в обьекте через одну переменную, вы меняете значение всех переменных ссылающихся на тот же обьект

    • Обьекты (object)
      25.08.2022

      Является ссылочным типом переменной

      const preson = {
      name:'Bob',
      age: 25;
      }
      const person2 = person;
      person2.age = 26
      console.log(person.age) //26
      console.log(person2.age) //26

      const preson = {
      name:'Bob',
      age: 25;
      }
      const person3 = Object.assign({}, person);
      person2.age = 26
      console.log(person.age) //25
      console.log(person2.age) //26

      const preson = {
      name:'Bob',
      age: 25;
      }
      const person3 = {... preson} // оператор "..." разделяет обьект на свойства, в данном случае мы разделяем обьект person на свойства нового обьекта, тем самым избавляемся от мутаций обьекта person при мутации обьекта person3
      person2.age = 26
      console.log(person.age) //25
      console.log(person2.age) //26

      В перечисленных вариантах, с оператором "..." и Object.assign, вложанные обьекты в обьекте person3 (если такие были бы) имеют общую ссылку на главный обьет, а не копируют его

      const preson = {
      name:'Bob',
      age: 25;
      }
      const person3 = JSON.parse(JSON.stringify(person)) // полное копирование с учетом вложенных обьектов в обьект person
      person2.age = 26
      console.log(person.age) //25
      console.log(person2.age) //26

Функции
25.08.2022
  • Функция возвращает undifiend если нет return
  • Аргументы функции передаются в вызове, а параметры в обьявлении функции
  • Внутри функци НЕ РЕКОМЕНДУЕТСЯ редактировать внешние обьекты
  • callback функция - функция которая передается аргументом в другую функцию
  • Во время вызова функции callback аргумент передается без - ()
  • Одна функция должна выполнять одну задачу
  • Переменные обьявленные в блоках ("{код блока}") имеют локальную область видимости данного блока
  • В случае указания значения переменной, без ее обьвяления, переменная создается автоматически в глобальной области видимости
Обьявление функции
02.09.2022

  • Анонимное обьявление

    function(){
    // тело функции
    }

    const name = function(){
    // тело функции
    }
    name();

  • Стрелочная функция

    Данная функция не привязана к собственному this, arguments, super, или new.target

    () => {
    // тело функции
    }
    // сокращенный вариант если у ункции один параметр a => {
    // тело функции
    }
    // сокращение если у тела функции одно выражение (a, b) => a + b

    const name = () => {
    // тело функции
    }
    name();

  • Классика

    function funcName(){
    // тело функции
    }
    funcName();

Операции
05.09.2022

  • in

    проверка наличия наследуемых свойств или методов обьекта или массива

    "x" in point // есть ли свойство x в обьекте point

    1 in point // есть ли элемент с id = 1

    toString in point // есть ли наследуемый метод в обьекте point

  • instanceof

    x instanceof Date // возвращает true если x наследуемый класс от Date

    x instanceof Number // true если x является экземляром Number

  • Что такое eval

Массивы
08.09.2022
  • push добавляет элемент в конец массивы
  • pop удаляет элемент массивы и возращает его значение. (Pop можно присвоить значению переменной)
  • unshift добавляет в начало массива
  • shift добавляет первый элемент в массиве и возвращает его
  • forEach, принимает в себя только кбэк функции(стрелочная или анонимная)
  • map перебирает все элементы массива и возвращает измененный массив формируя его исходя из результатов cback функции
  • const {name,age} = userObject; Деструктуризация обьекта - создает переменные name и age с значениями из user.name и user.age соотвественно
  • const [name,age] = userObject; Деструктуризация массива - создает переменные name и age с значениями из user[0] и user[1] соотвественно
